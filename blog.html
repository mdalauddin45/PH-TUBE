Discuss the scope of var, let, and const!

VAR :

  • Scope : var has function-level scope. It means that a variable declared with var is visible throughout the function in which it is declared.
  • Hoisting : Variables declared with var are hoisted to the top of their function or global scope. However, only the declaration is hoisted, not the initialization.
  • Reassignment : var allows for redeclaration and reassignment of variables.

LET :

  • Block Scope : let has block-level scope. It means that a variable declared with let is only accessible within the block it is declared in (if, for, while, etc.).
  • Hoisting : Variables declared with let are hoisted to the top of their block but are not initialized. Accessing the variable before its declaration results in a ReferenceError.
  • Reassignment : let allows reassignment but not redeclaration of the same variable within the same block scope.

CONST :

  • Block Scope : const also has block-level scope.
  • Hoisting : Like let, variables declared with const are hoisted to the top of their block but are not initialized. Accessing the variable before its declaration results in a ReferenceError.
  • Reassignment : Variables declared with const must be initialized during declaration and cannot be reassigned. However, for objects and arrays, while the reference cannot be changed, the internal properties or elements of the object/array can be modified.
Tell us the use cases of null and undefined!

null :

  • Intentional absence of value : It is a deliberate assignment to a variable or an object property to signify that it has no value or is explicitly empty.
  • Use cases :
    • Explicitly clearing values : Sometimes used to explicitly reset or clear a variable or an object property.
    • As a placeholder : Can be used as a placeholder until a value is assigned.
    • Checking for uninitialized values : It can be used to check if a value has been intentionally left blank or not assigned.

undefined :

  • Intentional absence of value : It typically indicates an uninitialized variable or a property that has not been assigned a value explicitly.
  • Use cases :
    • Variable declaration without initialization : When a variable is declared but not assigned any value, it automatically gets the value undefined.
    • Missing function arguments : If a function is called with fewer arguments than declared parameters, the remaining parameters will have an undefined value.
    • Property of an object that has not been set : When trying to access an object property that doesn't exist, it returns undefined.
What do you mean by REST API?

REST API :

An API, or application programming interface, is a set of rules that define how applications or devices can connect to and communicate with each other. A REST API is an API that conforms to the design principles of the REST, or representational state transfer architectural style. For this reason, REST APIs are sometimes referred to RESTful APIs.
